# 🌊 旅遊行程網站 — 可重複套用範本 (SOP)

> 這份是「花蓮三天兩夜」專案整理出來的做法，未來任何出遊都能照這流程做出一個
> **手機優先、可導航、可打卡拍照、能生成回顧影片、放上 GitHub 分享**的行程網站。
> 給 Claude / AI 助理看也行 —— 直接把這份貼給它，說「照這個範本幫我做 ○○ 旅遊」即可。

---

## 一、最終成品長怎樣（功能清單）

一頁式 `index.html`（HTML/CSS/JS 全內嵌，無需建置、無框架），含：

1. **Hero 標題區** — 行程名稱、日期、同行人數、住宿/交通標籤
2. **頂部 sticky 導覽 tab** — D1/D2/D3… ＋ 美食 ＋ 咖啡（scroll-spy 自動高亮）
3. **每日時間軸卡片** — 時間、地點、推薦菜、價格、Google★評分、公休、點對點距離
4. **每個地點按鈕** — Google 地圖導航 / Apple 地圖 / 📞一鍵撥號
5. **替代方案** — 每個正餐可展開 2–4 個備案（提高彈性）
6. **美食/咖啡口袋名單** — 分類（海鮮/親子/在地推薦/宵夜/夜市/甜點/咖啡…）
7. **打勾完成** — 點圈圈打勾，卡片收合成一行；存 localStorage；有進度條
8. **加照片打卡** — 每張卡可加照片（存 IndexedDB，可拍照或選相簿、多張）
9. **自由打卡** — 臨時去的店輸入店名＋照片
10. **回顧影片** — 照片做成幻燈片，可全螢幕播放、下載 .webm
11. **底部快捷列** — 快速跳各區
12. **必看提醒區** — 訂位、公休地雷、親子 tips、交通/保險
13. **無障礙** — 圖片 alt、prefers-reduced-motion、手機 375px 不破版

## 二、製作流程（10 步）

### Step 1. 收集需求（問使用者）
- 幾天幾夜、日期（換算成星期幾，公休判斷用）
- 同行人數（幾大幾小、有無小孩年齡 → 影響親子標註、低消、安全座椅）
- 交通方式（自駕/租車/火車）、住宿（查證地址電話入住退房）
- 主要行程錨點（賞鯨、景點、必去的店）
- 有沒有在地朋友推薦清單（最有價值，優先納入）

### Step 2. 查證資料（用搜尋工具，逐家確認）
每家店/景點都要查證並記錄：
- **正確地址、電話**（多來源交叉比對，官網優先）
- **Google 評分＋評論數**、價格帶/人均
- **營業時間＋公休日**（最容易踩雷，務必對照旅遊日期的星期）
- **推薦菜色/必點**、訂位方式（電話/線上/不可訂）
- 特殊規則（只收現金、低消、需提前多久訂）

> 🔑 重點：**公休日一定要對照實際日期**。例：週日去，標出哪些店週日休、提供備案。

### Step 3. 規劃動線
- 把地點按地理位置＋時間排序，算**點對點車程/距離**
- 正餐配「順路、當天有開」的店；早餐配「早開、出餐快」
- 趕的時段（如賞鯨報到）放快速的店，悠閒時段放需排隊的名店
- 每個正餐準備 2–4 個替代方案

### Step 4. 用 ui-ux-pro-max skill 生成設計系統（選用但推薦）
```
python ~/.claude/skills/ui-ux-pro-max/scripts/search.py "travel itinerary trip planner mobile family" --design-system
```
- 旅遊類建議：Sky blue + adventure orange 配色、playful 字體、Intersection Observer 動畫
- 中文網站字體用 **Noto Sans TC**（Fredoka 等不支援中文）
- 對照它的 Pre-Delivery Checklist 收尾

### Step 5. 套用 HTML 範本（見第四節骨架）
把查證好的資料填進卡片結構。

### Step 6. 本地測試（重要！）
```powershell
# 起本機 server（file:// 會被擋）
cd <專案目錄>; python -m http.server 8765
```
- 用 Playwright 或手機開 `http://localhost:8765/index.html`
- **檢查 console error = 0**（favicon 404 可忽略）
- 用手機尺寸（390px）截圖檢查：底部列橫向、按鈕不換行、無水平捲動
- 測打勾、加照片、回顧播放

### Step 7. 建 git repo + 推 GitHub
```powershell
cd <專案目錄>
git init -b main
git add -A
git commit -m "初版行程網站"
# 先到 https://github.com/new 建立 repo（Public、不要勾 README）
git remote add origin https://github.com/<帳號>/<repo名>.git
git push -u origin main   # 第一次會跳瀏覽器登入授權
```

### Step 8. 開 GitHub Pages
repo → Settings → Pages → Source: Deploy from a branch → main / root → Save
網址：`https://<帳號>.github.io/<repo名>/`

### Step 9. 寫 LINE 分享文字（給友人）
簡短版：網址 ＋ 3–4 句功能（切換每天、點地點導航、打勾、加照片回顧）。
⚠️ 提醒「打勾照片存各自手機、不互通」。

### Step 10. 旅程中／後持續更新
- 想加店：朋友傳訊 → 查證 → 加卡片 → `git push` → 友人重整網頁就更新
- 旅程結束：用「下載影片」生成回顧

## 三、踩過的雷（未來避免）

| 雷 | 解法 |
|---|---|
| 底部快捷列破版（圖示直向佔半螢幕）| `.quickbar` 內務必包一層 `.wrap`（flex 寫在 wrap 上）|
| 按鈕在窄手機換行、卡片被撐長 | 按鈕 `min-width:0; flex:1; white-space:nowrap`，電話縮成圖示 |
| 加照片只能拍照、不能選相簿 | input **不要**加 `capture` 屬性；加 `multiple` 可多選 |
| IndexedDB 初始化 | `open().then(db=>{ ... })` 記得把回傳的 db 存起來再用 |
| 替代餐廳/臨時店無法拍照 | 加「自由打卡」：輸入店名 + 照片，不綁卡片 |
| 公休日沒對照日期 | 每家標公休，並針對旅遊當天的星期特別提示 |
| file:// 無法測試 | 用 `python -m http.server` 起本機 server |
| 真 mp4 影片 | 純前端難穩定；改用 Canvas + MediaRecorder 輸出 .webm |
| emoji icon | 親友旅遊情境用 emoji OK（活潑）；正式 SaaS 才需換 SVG |

## 四、HTML 骨架（精簡版，可直接擴充）

> 完整實作參考同目錄的 `index.html`。核心結構：

```
<head> Noto Sans TC + <style>（CSS 變數配色系統）</head>
<body>
  <header class="hero"> 標題/日期/人數標籤 </header>
  <nav class="daynav"><div class="wrap"> D1 D2 D3 美食 咖啡 </div></nav>
  <main class="wrap">
    <div class="progress"> 完成度進度條 </div>
    <div class="recap-card"> 自由打卡/播放回顧/下載影片 </div>

    <section id="day1">
      <div class="day-head"> 日期膠囊 + 標題 </div>
      <div class="tl">  <!-- timeline -->
        <div class="item food|play|move">
          <div class="card">
            <span class="time">時間</span>
            <h3>店名 <span class="tag">標籤</span></h3>
            <div class="meta">★評分 · 價格 · 地址</div>
            <div class="rec">推薦菜</div>

            <div class="btns">Google / Apple / 📞</div>
```

## 五、關鍵技術備忘

- **存資料**：打勾用 `localStorage`、照片用 `IndexedDB`（容量大）。皆存本機、各裝置獨立。
- **照片壓縮**：上傳前用 canvas 縮到 ~1280px、JPEG 0.82，避免爆容量。
- **回顧影片**：`canvas.captureStream(30)` + `MediaRecorder`，輸出 `video/webm`。
  幻燈片用 `requestAnimationFrame` 畫淡入淡出 + 地點字幕。
- **導航連結**：
  - Google：`https://www.google.com/maps/search/?api=1&query=店名+地址`
  - Apple：`https://maps.apple.com/?q=店名+地址`
  - 電話：`tel:0XXXXXXXXX`
- **手機優先**：`max-width:560px` 置中容器、`viewport-fit=cover`、`env(safe-area-inset-*)`。
